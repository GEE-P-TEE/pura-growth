# Pura Growth - Master Plan

**Project Summary**: Pura Growth is a comprehensive web-based dashboard application designed exclusively for internal company use to manage and automate multi-platform growth marketing for supplement brands. The platform centralizes social media content management (30 posts/day across Instagram, TikTok, Twitter), giveaway campaigns (8-12 monthly), educational email sequences, and performance analytics with AI-powered content generation capabilities.

## App Overview and Objectives

Pura Growth is a comprehensive web-based dashboard application designed to streamline and automate multi-platform growth marketing for supplement brands. The platform serves as a centralized hub for managing social media content, giveaway campaigns, educational email sequences, and performance analytics across Instagram, TikTok, and Twitter.

### Primary Objectives
- Automate the planning and scheduling of 30 daily posts across three major social platforms
- Manage 8-12 monthly giveaway campaigns with automated lead capture workflows
- Create and deploy 8-day educational email courses with integrated landing pages
- Provide comprehensive analytics and performance tracking for all marketing activities
- Streamline content creation through AI-powered generation tools

## Target Audience

### Primary Users
- Internal company team members managing supplement brand campaigns
- Content creators and marketers within the organization
- Campaign analysts tracking performance metrics

### User Characteristics
- Marketing professionals with experience in social media management

## High-Level Technical Stack Recommendations

### Frontend Architecture
- **Framework**: Next.js 14 with App Router
- **Styling**: Tailwind CSS with shadcn/UI component library
- **State Management**: React Context API with custom hooks
- **Type Safety**: TypeScript for enhanced development experience

### Backend and Database
- **Backend-as-a-Service**: Supabase for database, authentication, and storage
- **Database**: PostgreSQL (via Supabase)
- **File Storage**: Supabase Storage for media assets
- **Real-time Features**: Supabase Realtime for live updates

### Authentication and Security
- **Authentication**: Email/password login with Supabase Auth
- **Authorization**: Role-based access control (single user type with full access)
- **Security**: Row-level security policies in Supabase

### External Integrations
- **AI Services**: OpenAI API, Gemini API, VEO3 API
- **Email Marketing**: Klaviyo API for email automation
- **Social Media**: Instagram API, Twitter API, TikTok API
- **Analytics**: Custom analytics engine with third-party integrations

# Pura Growth - App Flow, Pages and Roles

## User Roles and Access Rights

### Single User Type: Full Access Administrator
- **Role Name**: Growth Marketing Manager
- **Access Level**: Complete system access with full CRUD permissions
- **Capabilities**:
  - Create, edit, delete, and schedule all content across platforms
  - Manage all giveaway campaigns and email sequences
  - Access all analytics and reporting features
  - Configure integrations and system settings
  - Export data and generate custom reports
  - Manage AI content generation settings

## Core Application Pages

### 1. Dashboard Homepage (`/dashboard`)
**Function**: Central command center providing overview of all marketing activities
**Key Features**:
- Real-time performance metrics widget
- Content calendar preview (next 7 days)
- Active giveaway campaigns status
- Email course performance summary
- Recent social media engagement stats
- Quick action buttons for common tasks
- Notification center for alerts and milestones

**Function**: Comprehensive content creation and scheduling hub

## User Flow Navigation

### Primary User Journey: Content Creation to Publishing
1. **Entry Point**: Dashboard homepage (`/dashboard`)
2. **Content Planning**: Navigate to Content Calendar (`/content/calendar`)
3. **Content Creation**: Use AI Generator (`/content/ai-generator`) or Templates (`/content/templates`)
4. **Media Management**: Upload assets via Media Manager (`/content/media`)
5. **Scheduling**: Drag-and-drop content to calendar slots
6. **Performance Monitoring**: Track results in Analytics (`/analytics/social`)

### Secondary User Journey: Giveaway Campaign Management
1. **Campaign Setup**: Start from Dashboard or Campaigns (`/campaigns/create`)
2. **Template Selection**: Choose from Templates (`/campaigns/templates`)
3. **Keyword Configuration**: Set up triggers (`/campaigns/keywords`)
4. **Launch Campaign**: Activate and monitor from Active Campaigns (`/campaigns/active`)
5. **Performance Review**: Analyze results (`/campaigns/analytics`)

### Tertiary User Journey: Email Course Development
1. **Course Planning**: Begin at Email Courses (`/email-courses/sequences`)
2. **Sequence Building**: Create 8-day email series with drag-and-drop editor
3. **Landing Page Creation**: Design conversion pages (`/email-courses/landing-pages`)
4. **Integration Setup**: Connect with Klaviyo (`/integrations/email`)
5. **A/B Testing**: Optimize performance (`/email-courses/ab-testing`)
6. **Analytics Review**: Monitor course effectiveness (`/email-courses/analytics`)

## User Interaction Patterns

### Content Creation Flow
1. User selects content type (post, story, video)
2. AI generates initial content based on prompts
3. User edits and customizes generated content
4. User selects target platforms and scheduling
5. Content is queued for automated publishing

### Campaign Management Flow
1. User creates campaign using wizard interface
2. System generates keyword triggers and response templates
3. User customizes automation rules and lead magnets
4. Campaign launches with real-time monitoring
5. Performance data feeds back into analytics dashboard

### Email Course Development Flow
1. User outlines 8-day course structure
2. AI assists with email content generation
3. Landing pages are created with conversion optimization
4. Klaviyo integration handles subscriber management
5. A/B testing optimizes open rates and conversions

## Data Flow Between Pages

### Cross-Page Data Sharing
- Content created in `/content` appears in `/social` platform views
- Campaign performance from `/campaigns` feeds into `/analytics`
- Email course metrics from `/email-courses` integrate with `/analytics/email`
- AI-generated content from any section saves to respective template libraries
- User settings from `/settings` apply across all application pages

### Real-Time Updates
- Dashboard widgets update automatically from all system activities
- Calendar views reflect real-time scheduling changes
- Analytics dashboards refresh with live performance data
- Notification system provides instant alerts across all pages
